## 1. Назначение

`platformcat` — расширяемая платформа на базе Spring Boot. Сам Spring Boot
выступает платформой, поверх которой строится вся функциональность, а каждая
фича поставляется как отдельный подключаемый стартер (плагин).

### Что такое фича

**Фича** — это комплексное решение конкретной задачи целиком. Примеры:
основное хранение данных в БД, авторизация пользователя, проверка ACL объектов.

Не является фичей отдельное дополнение, изменение или частная функциональность.
Отдельные части и функциональности внутри фичи — это её составляющие, а не
самостоятельные фичи. Правило: **отдельным стартером оформляется именно фича**;
её внутренние части живут внутри этого же стартера.

Стартеры могут зависеть друг от друга — это нормально. Например, проверка ACL
объектов может зависеть от авторизации пользователя и от хранения данных в БД.

## 2. Цели и задачи

- Обеспечить модульную архитектуру, где функциональность добавляется без
  изменения ядра — подключением новых стартеров.
- Максимально опираться на встроенные механизмы Spring вместо самописных
  решений.
- Держать код в соответствии с принципами SOLID.

## 5. Нефункциональные требования

- **Платформа**: Java 25, Spring Boot 4.1.0 — платформа, на которой строится всё.
- **Архитектурные принципы**:
  - Код максимально соответствует принципам SOLID.
  - Каждая фича — самостоятельный стартер (плагин); ядро о конкретных фичах
    не знает.
  - Приоритет встроенным возможностям Spring над самописными решениями
    (например, для логики на событиях использовать шину событий Spring —
    `ApplicationEventPublisher` / `@EventListener`, а не собственный механизм).
- **Сборка**: Maven, многомодульная структура (модуль на стартер).
- **Логирование**: SLF4J + Logback, конфиг `logback-spring.xml`.
